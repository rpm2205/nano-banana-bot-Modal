# Описание продукта

Telegram-бот для персональной генерации фотореалистичных изображений людей.

Ключевая идея: **генерация по примеру (референсу) или по текстовому описанию с автоматическим сохранением идентичности пользователя** (лицо + внешность), без необходимости каждый раз переписывать промпты.

Бот ориентирован на:

- личное использование
- использование подписчиками Telegram-канала

---

# Основные сценарии использования

1. Пользователь один раз настраивает профиль (фото + внешность).
2. Для каждой генерации:
   - выбирает режим (по примеру или по описанию)
   - при необходимости добавляет уточнения
   - выбирает параметры генерации (формат, качество, количество вариантов)
3. Получает результат и может:
   - повторить генерацию
   - посмотреть и скопировать полный промпт

---

# Главное меню бота

```
👤 Мой профиль
🖼 Генерация по примеру
✍️ Генерация по описанию
```

---

# 👤 Мой профиль

Профиль используется **по умолчанию** во всех генерациях.

## Фото профиля

- Загрузить фото
- Заменить фото
- Посмотреть текущее фото

Фото хранится и используется для всех генераций, если пользователь не выбрал разовую генерацию.

## Параметры внешности (сохраняются)

### Глаза

- Выбор из списка:
  - голубые
  - серо-голубые
  - зелёные
  - карие
  - чёрные
- Возможность ввести свой вариант текстом

### Волосы

- Цвет:
  - блонд
  - русые
  - каштановые
  - чёрные
  - рыжие
  - мелированные
  - свой вариант (текст)
- Длина:
  - каре
  - средние
  - длинные
  - очень длинные
  - свой вариант (текст)

# 🖼 Генерация по примеру (референсу)

## Шаг 1. Использование профиля

```
Использовать сохранённый профиль?
✅ Да
➕ Нет, разовая генерация
```

Если выбрано «Нет»:

- пользователь загружает фото
- текстом вводит параметры глаз и волос (разово, не сохраняются)

## Шаг 2. Загрузка референса

Пользователь загружает изображение (например, из Pinterest), которое задаёт стиль и композицию.

## Шаг 3. Дополнительные уточнения (необязательно)

Пользователь может текстом добавить пожелания (улыбка, поза, настроение, детали сюжета и т.д.).

## Шаг 4. Параметры генерации (выбираются внутри сценария)

### Формат изображения

- 9:16 — сторис
- 3:4 — пост
- 1:1 — квадрат

### Качество

- 1K
- 2K

### Количество вариантов

- 1

Значения по умолчанию:

- 9:16
- 2K
- 1 вариант

## Шаг 5. Генерация

Процесс:

1. LLM анализирует референс и формирует текстовое описание стиля.
2. Бот автоматически модифицирует промпт:
   - добавляет запрет на изменение внешности
   - добавляет параметры глаз и волос
   - добавляет технические параметры (формат, качество)
   - добавляет пользовательские уточнения
   - при необходимости убирает дубли в требованиях в пользу тех, что выбрал пользователь
3. Итоговый промпт отправляется в модель генерации изображений.

## Результат

Пользователь получает изображение(я) и кнопки:

```
🔁 Повторить (новый вариант)
📋 Показать промпт
```

---

# ✍️ Генерация по описанию (по промпту)

Сценарий аналогичен генерации по примеру, но без референса.

Шаги:

1. Использовать профиль? Да / Нет
2. Ввод текстового описания (необязательно)
3. Выбор параметров генерации (формат, качество, количество)
4. Генерация
5. Результат + кнопки:
   - 🔁 Повторить
   - 📋 Показать промпт

---

# Промпты

- Промпт всегда формируется **прозрачно**.
- Пользователь может посмотреть и скопировать **полный итоговый промпт**, использованный для генерации конкретного изображения.
- Бот должен присылать промпт в сворачивающемся блоке (станадртная функция Telegram), чтобы он не занимал много места. Пользователь при необходимости его развернет и прочитает/скопирует.

---

# Seed и повторяемость

- Seed генерируется ботом автоматически.
- Пользователь не взаимодействует с seed напрямую.
- Кнопка «Повторить» запускает генерацию с новым seed.

---

# История

Отдельный раздел истории или галереи **не предусмотрен**.

Пользователь использует стандартную историю чата и раздел «Медиа» Telegram.

  - `image_size`: 1K, 2K
  - `aspect_ratio`: 1:1, 3:4, 4:3, 9:16, 16:9

# FSM (Finite State Machine)

## Что такое FSM

FSM (Finite State Machine) — это **конечный автомат состояний**. В контексте Telegram-бота это строгая схема:

- в каком состоянии сейчас находится пользователь
- какое действие от него ожидается
- какое действие допустимо
- в какое состояние бот переходит дальше

FSM нужен, чтобы:
- сценарии не ломались
- бот не путался в шагах
- можно было безопасно масштабировать логику
- исключить хаос из «если пользователь вдруг написал не туда»

FSM — это не про UI, а про **железную логику диалога**.

---

## FSM: глобальные состояния

### IDLE
Начальное состояние.

Допустимые действия:
- открыть главное меню
- выбрать пункт меню

Переходы:
- IDLE → PROFILE_EDIT
- IDLE → GEN_BY_REFERENCE_START
- IDLE → GEN_BY_TEXT_START

---

## FSM: редактирование профиля

### PROFILE_EDIT

Подсостояния:
- PROFILE_PHOTO_WAIT
- PROFILE_EYES_WAIT
- PROFILE_HAIR_COLOR_WAIT
- PROFILE_HAIR_LENGTH_WAIT

Переходы:
- PROFILE_EDIT → IDLE (после сохранения)

---

## FSM: генерация по примеру (референсу)

### GEN_BY_REFERENCE_START

Шаг 1. Использовать профиль?

Переходы:
- → GEN_BY_REFERENCE_PROFILE_YES
- → GEN_BY_REFERENCE_PROFILE_NO

---

### GEN_BY_REFERENCE_PROFILE_NO

Ожидание:
- загрузка фото
- ввод глаз / волос текстом

После получения → GEN_BY_REFERENCE_REF_WAIT

---

### GEN_BY_REFERENCE_REF_WAIT

Ожидание:
- загрузка референса

Переход → GEN_BY_REFERENCE_HINTS_WAIT

---

### GEN_BY_REFERENCE_HINTS_WAIT

Ожидание:
- текстовые уточнения
- или команда «пропустить»

Переход → GEN_BY_REFERENCE_PARAMS_WAIT

---

### GEN_BY_REFERENCE_PARAMS_WAIT

Выбор параметров:
- формат
- качество
- количество вариантов

Переход → GEN_BY_REFERENCE_GENERATE

---

### GEN_BY_REFERENCE_GENERATE

Действия:
1. Вызов LLM для описания референса
2. Fallback при ошибках
3. Сбор итогового промпта
4. Генерация изображения

Переходы:
- → GEN_BY_REFERENCE_RESULT
- → GEN_ERROR (если все fallback исчерпаны)

---

### GEN_BY_REFERENCE_RESULT

Состояние результата.

Доступные действия:
- 🔁 Повторить (возврат в GEN_BY_REFERENCE_GENERATE с новым seed)
- 📋 Показать промпт
- Вернуться в IDLE

---

## FSM: генерация по описанию

### GEN_BY_TEXT_START

Шаги:
1. Использовать профиль? Да / Нет
2. Ввод текстового описания
3. Выбор параметров генерации
4. Генерация

Подсостояния:
- GEN_BY_TEXT_PROFILE_CHECK
- GEN_BY_TEXT_PROMPT_WAIT
- GEN_BY_TEXT_PARAMS_WAIT
- GEN_BY_TEXT_GENERATE
- GEN_BY_TEXT_RESULT

Логика аналогична сценарию по примеру, без шага анализа изображения.

---

## FSM: ошибки

### GEN_ERROR

Условия:
- основная модель недоступна
- все fallback-модели недоступны

Поведение:
- пользователь получает понятное сообщение
- предлагается повторить позже

После → IDLE

# Схема данных (Data Model)

Ниже описана минимально достаточная схема данных для реализации бота с учётом FSM, мультипользовательского режима и масштабирования.

---

## User

Хранит информацию о пользователе Telegram.

**Поля:**
- `id` — внутренний ID
- `telegram_id` — Telegram user ID
- `username` — Telegram username (опционально)
- `created_at`
- `is_active`
- `daily_generation_count`
- `daily_generation_limit`

---

## UserProfile

Профиль пользователя, используемый по умолчанию во всех генерациях.

**Поля:**
- `id`
- `user_id` → User.id
- `photo_id` — ссылка на сохранённое фото лица (Telegram file_id или storage key)
- `eyes_color` — строка
- `hair_color` — строка
- `hair_length` — строка
- `updated_at`

Ограничение:
- один активный профиль на пользователя

---

## TempProfile (разовая генерация)

Используется, если пользователь выбрал генерацию **не по профилю**.

**Поля (хранятся временно, в рамках FSM):**
- `photo_id`
- `eyes_color`
- `hair_color`
- `hair_length`

После завершения сценария — удаляется.

---

## GenerationRequest

Описывает одну попытку генерации (один сценарий).

**Поля:**
- `id`
- `user_id`
- `mode` — `reference` | `text`
- `used_profile` — `profile` | `temp`
- `reference_image_id` — если mode = reference
- `user_prompt` — текстовые уточнения пользователя
- `generated_prompt` — итоговый промпт (сохранён полностью)
- `format` — 9:16 | 3:4 | 1:1
- `quality` — 1K | 2K
- `variants_count`
- `seed`
- `status` — pending | processing | completed | error
- `created_at`

---

## GeneratedImage

Результат генерации изображения.

**Поля:**
- `id`
- `generation_request_id`
- `image_file_id` — Telegram file_id или storage key
- `model_used`
- `seed`
- `created_at`

Один GenerationRequest → 1..N GeneratedImage (в зависимости от variants_count).

---

## ModelFallbackLog

Лог попыток вызова моделей (для дебага и контроля расходов).

**Поля:**
- `id`
- `generation_request_id`
- `model_name`
- `stage` — prompt_generation | image_generation
- `status` — success | error
- `error_message` (опционально)
- `created_at`

---

## FSMState

Хранит текущее состояние пользователя в FSM.

**Поля:**
- `user_id`
- `state`
- `state_payload` — JSON (временные данные шага)
- `updated_at`

---

# Принципы хранения данных

- Профиль пользователя хранится постоянно
- TempProfile и FSMState — временные сущности
- Все итоговые промпты сохраняются (прозрачность + воспроизводимость)
- Изображения хранятся минимально (Telegram file_id предпочтительнее)
- История UI не дублируется — используется история чата Telegram

---

# Ключевой принцип продукта

**Минимум действий → максимальный контроль → воспроизводимый результат.**

Схема данных поддерживает:
- FSM без хаоса
- fallback моделей
- повторяемость генераций
- масштабирование под публичное использование
