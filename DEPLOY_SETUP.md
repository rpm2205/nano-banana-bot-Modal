# Настройка автоматического деплоя в Modal из GitHub

## Описание
После настройки при каждом коммите в ветку `main` приложение будет автоматически деплоиться в Modal.

## Шаги настройки

### 1. Создание токена в Modal

1. Перейдите на https://modal.com/settings/tokens
2. Нажмите "Create token" или "New token"
3. **Важно:** Скопируйте **Token ID** и **Token Secret** точно, без лишних пробелов в начале или конце
4. Токены обычно выглядят как длинные строки символов (например, начинаются с `ak-` для Token ID)

### 2. Добавление секретов в GitHub

1. Откройте ваш репозиторий на GitHub
2. Перейдите в **Settings** → **Secrets and variables** → **Actions**
3. Нажмите **New repository secret**
4. Добавьте два секрета:
   - **Name**: `MODAL_TOKEN_ID`
     **Value**: ваш Token ID из Modal (скопируйте точно, без пробелов)
   - **Name**: `MODAL_TOKEN_SECRET`
     **Value**: ваш Token Secret из Modal (скопируйте точно, без пробелов)

**⚠️ Важно при копировании токенов:**
- Копируйте токены точно, без лишних пробелов в начале или конце
- Не добавляйте кавычки вокруг токенов
- Если токен показан в несколько строк, скопируйте его полностью как одну строку

*Примечание: Workflow автоматически удаляет пробелы из токенов, но лучше копировать их правильно с самого начала.*

### 3. Проверка работы

После настройки:
1. Сделайте коммит в ветку `main` (или создайте Pull Request и смержите его)
2. Перейдите в раздел **Actions** вашего репозитория на GitHub
3. Вы увидите запущенный workflow "Deploy to Modal"
4. После успешного выполнения приложение будет задеплоено в Modal

## Как это работает

Workflow файл находится в `.github/workflows/deploy.yml` и выполняет следующие действия:

1. ✅ Проверяет код из репозитория
2. ✅ Устанавливает Python 3.11 (с кэшированием для ускорения)
3. ✅ Устанавливает Modal CLI (пакеты кэшируются между запусками)
4. ✅ Аутентифицируется в Modal используя секреты из GitHub
5. ✅ Выполняет `modal deploy main.py`

**Примечание:** GitHub Actions создает свежий виртуальный образ при каждом запуске, поэтому установка происходит каждый раз. Однако благодаря кэшированию pip-пакетов повторная установка Modal CLI происходит быстрее (обычно несколько секунд вместо ~10-15 секунд).

## Устранение проблем

### Ошибка "Token ID is malformed"

Если вы видите эту ошибку при деплое:

1. **Проверьте токены в GitHub Secrets:**
   - Откройте Settings → Secrets and variables → Actions
   - Убедитесь, что токены скопированы полностью и правильно

2. **Пересоздайте токены в Modal:**
   - Перейдите на https://modal.com/settings/tokens
   - Создайте новый токен
   - Скопируйте Token ID и Token Secret точно, без пробелов
   - Обновите секреты в GitHub

*Примечание: Workflow автоматически удаляет пробелы из токенов, но если ошибка повторяется, проверьте, что токены скопированы полностью.*

## Отключение автоматического деплоя

Если нужно временно отключить автоматический деплой:
- Удалите или переименуйте файл `.github/workflows/deploy.yml`
- Или закомментируйте секцию `on:` в файле workflow
